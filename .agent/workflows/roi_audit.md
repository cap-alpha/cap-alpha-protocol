# /roi_audit Workflow (Productivity Tracking Mandate)

Whenever you complete a major task, or whenever you are processing a new `/todo`, you MUST silently execute the ROI Audit to ensure we are maximizing the value of the Antigravity quota and minimizing 'productivity illusion'.

1. **Leverage Check:** Analyze the current active task against the core Monetization and Growth objectives. Does this increase revenue, attract 'sports personalities', or harden production?
2. **The "Wait, Should We?" Protocol:** If a task requires more than 5 complex sequential tool calls and does not directly contribute to the Monetization Engine or System Uptime, STOP. Do not bleed quota. Notify the user: *"Is this micro-optimization exactly what you want to spend our quota on right now?"*
3. **Execution Efficiency / Quota Defense:** 
   - In any Python scripts or test suites, fail fast. Do not run 10-minute tests blindly. Use limits, small batches, and mock local paths before committing to full execution to save quota/time.
   - Summarize code changes in simple diffs rather than massive rewrites when possible.
4. **Mandate Enforcement in `/todo`:** Automatically apply this filter to any new tasks added via `/todo`. Push tasks that yield the highest business impact to the top of the backlog, and push non-critical formatting/styling to the bottom.
