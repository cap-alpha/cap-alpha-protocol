# NFL Dead Money - Pipeline Hardening & 2026 Updates

The Medallion Architecture data pipeline has been fully refactored, resolving schema type conflicts, string/syntax errors in Gold-level BigQuery SQL generations, and ensuring immutable ingestion. The pipeline correctly hydrated the `cap-alpha-protocol` BigQuery tables.

## Pipeline Changes overview

1. **BigQuery Schema Dialect Fixed**: Eliminated all `VARCHAR` types generated by Python abstractions, moving entirely to `STRING`. Additionally modified integer conversions mapping strictly to `INT64` and float conversions mapping to `FLOAT64` where previously standard SQL `FLOAT`/`BIGINT` resulted in ingestion failures. Added the missing `week` parameter to `silver_pfr_game_logs`.
2. **BigQuery Operational Logic Resync**: Refactored the data-munging queries inside the Python pipeline script. `TRY_CAST()` functions were converted to BigQuery's `SAFE_CAST()`, modulo operators like `%` were correctly translated to `MOD()`, and unsafe standard string concatenations like `||` were swapped for native `CONCAT()`. 
3. **Pushed Fresh Data**: Scraped the brand-new 2026 Spotrac equivalent layout (via OverTheCap base endpoints to bypass 403 blocks), parsing total guarantees, positional metrics, and fresh team assignments properly to Bronze, Silver, and Gold.
