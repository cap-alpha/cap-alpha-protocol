# SP18.5-1: Silver Layer Audit — Schema Update Requirements

**Date**: 2026-04-24
**Sprint**: 18.5 — High-Frequency Silver Data Model Redesign
**GH Issue**: #64

---

## Goal

Audit every Silver layer table. Document current schemas, identify gaps against SCD Type 2
best practices, and list concrete `ALTER TABLE` / migration actions needed to support
high-frequency (hourly/daily delta) ingest without history loss.

---

## Canonical SCD Type 2 Columns (Target Standard)

Every Silver table that tracks mutable state must carry these columns:

| Column | Type | Constraint | Purpose |
|---|---|---|---|
| `valid_from` | `TIMESTAMP` | `NOT NULL` | When this row version became effective |
| `valid_until` | `TIMESTAMP` | nullable | `NULL` = currently active record |
| `is_current` | `BOOL` | `NOT NULL` | Partition-filter shortcut; mirrors `valid_until IS NULL` |
| `source_name` | `STRING` | `NOT NULL` | Data provider identifier (e.g. `spotrac`, `sportsdataio`) |
| `system_ingest_ts` | `TIMESTAMP` | `NOT NULL` | Wall-clock time the pipeline wrote this row |

Reference tables (rarely changing: teams, positions) only need `system_ingest_ts`.

### 2. `silver_player_metadata` — Player Bio (BigQuery, Active)

**Source**: `pipeline/src/silver_sportsdataio_transform.py` → `run_silver_transformation()`

**Current columns** (from transform code):

| Column | Type | Notes |
|---|---|---|
| `full_name` | `STRING` | |
| `birth_date` | `STRING` | Stored as `%Y-%m-%d` string — should be `DATE` |
| `college` | `STRING` | |
| `experience_years` | `INT64` | |
| `draft_round` | `INT64` | 0 = undrafted |
| `draft_pick` | `INT64` | 0 = undrafted |
| `team` | `STRING` | Current team (mutable — not tracked historically) |
| `position` | `STRING` | Current position (mutable) |
| `photo_url` | `STRING` | |
| `status` | `STRING` | Active / Inactive / etc. |

**Write mode**: `WRITE_TRUNCATE` — full overwrite on every pipeline run. **All history is lost.**

**Status**: No SCD columns at all. No surrogate key. No link to `silver_spotrac_contracts.player_id`.

**Required changes**:
1. Add SCD Type 2 columns: `valid_from`, `valid_until`, `is_current`, `source_name`, `system_ingest_ts`
2. Change write mode from `WRITE_TRUNCATE` to append via `execute_scd2_merge()` pattern
3. Fix `birth_date` type: store as `DATE` not `STRING`
4. Add `player_id` surrogate key (MD5 of `full_name`) to enable joins with `silver_spotrac_contracts`
5. Add `sportsdataio_player_id STRING` — preserve the vendor's native ID for stable cross-table references

**Migration**: `009_add_scd2_to_silver_player_metadata.sql`

## Cross-Cutting Issues

1. **`player_id` instability**: MD5 of `player_name` is the only cross-table join key. Common names cause collisions. Long-term fix: adopt a stable vendor ID (e.g. SportsData.io `PlayerID`) as the canonical `player_id`.

2. **No schema registry**: `silver_sportsdataio_transform.py:65` mentions a `schema.yaml` that does not exist. Should be created at `pipeline/config/silver_schema.yaml`.

3. **DuckDB remnants**: `financial_ingestion.py` and `strategic_engine.py` use raw DuckDB connections. These must be migrated to `DBManager` before SP18.5-2 (event-driven micro-batch) can be implemented.

4. **Write mode inconsistency**: `silver_spotrac_contracts` uses SCD2 append; `silver_player_metadata` uses WRITE_TRUNCATE. All Silver tables should use consistent append-only SCD2 patterns.
